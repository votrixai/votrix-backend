# 文案模板

---

## 默认风格：幽默搞笑

**所有文案默认采用幽默搞笑语气。** 除非 `marketing-context.md` 的 `## 品牌语气` 另有明确指定，否则一律按此执行：

- **语气：** 轻松、自嘲、有梗、不正经但不低俗
- **Hook：** 优先用反转、夸张、吐槽、自嘲开头，让人忍不住笑
- **正文：** 用口语化表达，可以用表情符号，避免正式商业腔
- **CTA：** 带点俏皮感，不要死板命令式
- **避免：** 过度自夸、说教、官方发言体

---

## Hook 公式

Hook 是帖子第一句，决定用户是否继续读。

### 幽默/搞笑型（默认优先使用）
- 「[自嘲式开场]，但我们 [意外的正面结果] 😅」
- 「我们本来只想 [小目标]，结果 [夸张后果]——」
- 「老实说：[行业常见吹牛话] 我们从不说。我们只说 [真实反转]。」
- 「[常见痛点] 吧？对，我们也是。然后我们做了这个 👇」
- 「当 [场景] 的时候你会怎么办？我们的答案可能有点离谱——」
- 「没想到吧，[出乎意料的事实]。你以为我们在 [误导性描述]，实际上——」

### 好奇心型
- 「你可能不知道，[常见认知] 其实是错的。」
- 「[令人意外的结果]——我们只用了 [时间]。」
- 「为什么 [行业] 里的人从不告诉你 [真相]？」

### 故事型
- 「上周发生了一件事，让我重新思考 [话题]。」
- 「3 年前我们 [过去状态]，今天 [现在状态]。」
- 「我差点犯了一个很多人都会犯的错误——」

### 价值型
- 「[数字] 个方法，帮你 [解决问题]（不需要 [常见痛点]）：」
- 「如何在 [时间] 内做到 [目标]：」
- 「停止 [错误做法]，改做这个：」

### 观点型
- 「我认为 [行业常见观点] 是被高估的。」
- 「大多数 [目标受众] 都在做一件浪费时间的事。」
- 「[热门话题] 没有你想的那么重要——这才是关键：」

## 结构模板

### 产品推广（Promotional）

```
[Hook — 聚焦客户痛点，不直接推产品]

[正文 — 描述问题，引出解决方案]

[产品介绍 — 1–2 句，简洁]

[社会证明 — 可选，数据或客户评语]

[CTA — 明确的下一步]

[Hashtag]
```

### 行业知识（Educational）

```
[Hook — 承诺价值或提出问题]

[核心内容 — 列表或步骤，3–5 点]

[总结或关键结论]

[CTA — 「保存这篇」「分享给需要的人」]

[Hashtag]
```

### 幕后故事（Behind-the-Scenes）

```
[Hook — 真实场景开头，有画面感]

[故事展开 — 过程、挑战、细节]

[结论或感悟]

[CTA — 「你们有类似经历吗？」「评论区聊聊」]

[Hashtag]
```

### 客户案例（Social Proof）

```
[Hook — 用数字或结果开头]

[客户背景 — 一句话，谁 / 什么情况]

[问题 → 解决过程 → 结果]

[CTA — 「想了解更多？主页链接」]

[Hashtag]
```

### 促销/限时（Promo）

```
[Hook — 制造紧迫感或惊喜感]

[优惠内容 — 清晰说明，几句话]

[限制条件 — 时间 / 数量 / 对象]

[CTA — 强，行动导向]

[Hashtag — 减少，不稀释 CTA 注意力]
```

### 互动/问答（Engagement）

```
[Hook — 问题开头，直接引发思考]

[背景或观点 — 给出你的立场]

[邀请互动 — 开放式问题]

[Hashtag]
```

---

## CTA 选项

根据帖子目标选择：

| 目标 | CTA 示例 |
|---|---|
| 引流到主页/网站 | 「主页链接，点击了解更多」「Bio 里有详细介绍」 |
| 增加互动 | 「你怎么看？评论告诉我们」「分享给需要的朋友」 |
| 增加关注 | 「关注我们，每周都有这类内容」 |
| 收集 leads | 「私信我们，发你完整资料」「留言『我要』，我们联系你」 |
| 促成购买 | 「限时优惠，今天截止」「扫码/点链接下单」 |
