# Twitter / X 内容规格

---

## 内容类型总览

| 类型 | 适合场景 | 触达模式 |
|---|---|---|
| Single Tweet | 快速公告、一句观点、短消息 | 粉丝 feed，互动则扩散 |
| Thread | 深度内容、教程、故事、多步骤 | 粉丝 feed，第一条决定展开率 |

---

## Single Tweet（单条推文）

**适用：** 快速公告、有力观点、活动提醒、互动问题

**字数限制：**
- 最多 280 字
- 链接固定占 23 字（无论链接长短）
- 图片不占字数

**文案结构：**
```
[正文 — 直接切入，不废话]
[链接（如有）]
[Hashtag — 1–2 个，太多显 spammy]
```

**媒体规格：**
- 图片：最多 4 张，16:9（1200×675）效果最好，JPG/PNG，最大 5MB
- GIF：最大 15MB
- 视频：最长 2 分 20 秒，MP4

**写作要点：**
- 每个字都要值得，删掉所有不必要的词
- 强观点 > 中性陈述，有态度才有互动
- 链接放末尾，缩短版链接或直接贴原链接均可（都占 23 字）
- Hashtag 最多 1–2 个，放末尾；Twitter Hashtag 效果比 IG 弱得多
- 问句式结尾互动率最高（「你怎么看？」「有同感的 RT」）

**配图 aspect_ratio：** `"16:9"`

---

## Thread（推文串）

**适用：** 深度内容展开、教程步骤、故事叙述、观点论证

**结构规则：**
- 每条推文独立成立，单独看也有价值
- 第一条是最关键的 — 决定用户是否展开整个 thread
- 用「/」或编号标注（可选）：「1/7」「2/7」
- 最后一条写总结 + CTA

**文案结构：**
```
[第 1 条 — hook，独立成立，有悬念或价值承诺]
配图（可选，第一条带图互动率更高）

[第 2–N 条 — 每条一个核心点，不超过 280 字]
简洁，每条之间有逻辑递进关系

[最后一条 — 总结 + CTA]
「关注我获取更多这类内容」
「转发这条帮助更多人」
「完整版链接在这里：[链接]」
```

**agent 生成 Thread 时按条拆分输出：**
```
---第 1 条---
[文案，≤280 字，含 hashtag 1–2 个]

---第 2 条---
[文案，≤280 字]

...

---第 N 条（结尾）---
[总结 + CTA]
```

**建议条数：**
- 3–5 条：轻量教程、观点论证
- 5–8 条：深度教程、完整故事
- 超过 8 条会掉读者，除非内容极有价值

**写作要点：**
- 不要用 thread 来凑字数，每条必须有独立价值
- 第一条结尾可加「👇」或「/thread」引导向下读
- Thread 比 single tweet 更适合深度内容，但需要更多 admin 编辑确认
- 发完 thread 后，把第一条 tweet 的链接固定在主页（pin）效果更好

---

## 最佳发布时间

| 类型 | 推荐时间 |
|---|---|
| Single Tweet | 周一–周五，早 8 点 或 下午 4–5 点 |
| Thread | 周一–周三，早 8–9 点（需要用户有时间展开读） |

---

## 草稿格式

### Single Tweet
```markdown
## 文案
[推文正文，≤280 字，链接占 23 字]

## 配图路径
[public_url，aspect_ratio: "16:9"，最多 4 张；与视频二选一，无则留空]

## 视频路径
[可选，admin 提供的视频 public_url，MP4，≤2分20秒；与配图二选一，无则留空]
```

### Thread
```markdown
## Thread 拆分

---第 1 条---
[文案，≤280 字 — hook，独立成立]
[配图 public_url，无则留空]

---第 2 条---
[文案，≤280 字]

---第 3 条---
[文案，≤280 字]

...

---最后一条---
[总结 + CTA，≤280 字]
```
