# Instagram 内容规格

---

## 内容类型总览

| 类型 | 适合场景 | 触达模式 |
|---|---|---|
| Single Feed | 常规内容、活动公告、产品展示 | 粉丝 feed |
| Carousel | 教程、对比、产品系列、步骤拆解 | 粉丝 feed，互动率高 |
| Reels | 品牌曝光、触达新用户、短视频内容 | 算法推流，可触达非粉丝 |
| Story | 限时活动、投票互动、幕后花絮 | 粉丝 story 栏，24h 消失 |

---

## Single Feed（单图 / 单视频）

**适用：** 产品推广、活动通知、行业知识、客户案例

**图片规格：**
- 比例：1:1（1080×1080）或 4:5（1080×1350，占屏更多）
- 格式：JPG / PNG，最大 8MB

**视频规格：**
- 时长：3s–60s
- 比例：1:1 / 4:5 / 9:16
- 格式：MP4，最大 100MB

**文案结构：**
```
[Hook — 第一句，决定是否点「更多」，125字以内]

[正文 — 品牌语气，具体有料，不空泛]

[CTA — 明确下一步]

[Hashtag — 10–15 个，放正文末尾或第一条评论]
```

**写作要点：**
- 前 125 字是显示区，之后折叠，hook 在这里
- 善用换行，避免大段密集文字
- Emoji 最多 3–5 个
- 链接不可点击，改写为「链接在主页 Bio」

**配图 aspect_ratio：** `"1:1"` 或 `"4:5"`

---

## Carousel（多图轮播，2–10 张）

**适用：** 步骤教程、产品对比、系列展示、知识卡片

**规格：**
- 张数：2–10 张
- 每张图片格式同 Single Feed
- 所有图片必须同一比例（建议 1:1）

**文案结构：**
```
[Caption — 只有一段，引导用户滑动]
通常写：「左滑查看全部 →」或「滑动看完整步骤」
第一张图的视觉内容即 hook，不依赖 caption

[Hashtag — 10–15 个]
```

**agent 需要额外生成「分页大纲」：**
```
第 1 张：[封面 — hook 标题，抓注意力]
第 2 张：[观点 / 步骤 1]
第 3 张：[观点 / 步骤 2]
...
最后一张：[CTA 页 — 明确下一步，品牌 logo]
```
每张只写一个核心点，配合图片视觉，文字不超过 15 字。

**配图 aspect_ratio：** `"1:1"`（多张保持统一）

---

## Reels（短视频）

**适用：** 品牌曝光、触达新用户、轻松有趣内容、产品使用展示

**视频规格：**
- 比例：9:16（1080×1920），竖屏必须
- 时长：15s / 30s / 60s / 90s（30–60s 算法推流效果最好）
- 格式：MP4

**文案结构：**
```
[Caption — 简短，30 字以内]
Reels 靠视频前 3 秒留住用户，caption 辅助

[Hashtag — 3–5 个即可，不需要堆]
```

**写作要点：**
- Caption 简短，核心信息在视频里传达
- 不需要放完整文案，可只放一句 hook + hashtag
- 链接写「链接在主页 Bio」

**配图 aspect_ratio：** `"9:16"`（封面图）

---

## Story（限时动态）

**适用：** 限时活动、投票互动、幕后花絮、快速公告

**规格：**
- 比例：9:16（1080×1920）
- 上下各留 14% safe zone（顶部被 UI 遮挡，底部被互动区遮挡）
- 持续时间：24 小时后自动消失
- 图片最长显示 7s，视频最长 15s

**无持久 Caption**，信息靠图片 / 视频本身传达。

**agent 需要生成：**
```
视觉内容描述：[画面主体 — 要在 safe zone 内]
文字叠加（如有）：[简短标语，最多 2 行]
互动贴纸建议：
  - Poll：[问题] / [选项 A] vs [选项 B]
  - Question：[提问文字]
  - Quiz：[问题] + [正确答案] + [干扰选项]
  - Countdown：[活动名称] [截止时间]
  - 无贴纸（纯展示）
```

**写作要点：**
- Story 不适合放核心长期内容，适合即时性、互动性内容
- 关键文字 / logo 放中间 72%，避免 safe zone 区域
- 有互动贴纸的 Story 互动率显著更高

**配图 aspect_ratio：** `"9:16"`

---

## 最佳发布时间

| 类型 | 推荐时间 |
|---|---|
| Feed / Carousel | 周二、三、五，早 9 点 或 晚 7–9 点 |
| Reels | 周一–周五，下午 3–5 点（算法推流，时间敏感度低） |
| Story | 早 8–9 点（上班前）或 晚 8–10 点（睡前刷手机） |

## 草稿格式

### Single Feed
```markdown
## 文案
[正文]

## Hashtag
[10–15 个]

## 链接
[无则留空]

## 配图路径
[image_generate 返回的 public_url，aspect_ratio: "1:1" 或 "4:5"]
```

### Carousel
```markdown
## 文案
[引导滑动的 caption，如「左滑查看全部 →」]

## 分页大纲
第 1 张：[封面标题 — hook]
第 2 张：[观点 / 步骤 1]
第 3 张：[观点 / 步骤 2]
...
最后一张：[CTA + 品牌]

## Hashtag
[10–15 个]

## 配图路径
[封面图 public_url，aspect_ratio: "1:1"]
```

### Reels
```markdown
## 文案
[简短 caption，30 字以内]

## Hashtag
[3–5 个]

## 视频路径
[admin 提供的视频 public_url，MP4，9:16；待视频时留空]

## 封面图路径
[可选，image_generate 返回的 public_url，aspect_ratio: "9:16"；不需要则留空]
```

### Story
```markdown
## 视觉内容
[画面描述，关键元素放中间 72%，safe zone 上下各 14% 留空]

## 文字叠加
[简短标语，最多 2 行，无则留空]

## 互动贴纸
[Poll: 问题 / 选项A vs 选项B]
或 [Question: 提问文字]
或 [Countdown: 活动名 / 截止时间]
或 [无]

## 配图路径
[public_url，aspect_ratio: "9:16"]
```
