# LinkedIn 内容规格

---

## 内容类型总览

| 类型 | 适合场景 | 触达模式 |
|---|---|---|
| Text Post | 观点输出、行业洞察、个人故事 | 粉丝 feed，互动高则算法扩散 |
| Image Post | 产品展示、活动回顾、数据可视化 | 粉丝 feed |
| Document Carousel | 教程、行业报告、知识卡片、品牌介绍 | 粉丝 feed，停留时长高 |
| Video | 产品演示、案例分享、团队故事 | 粉丝 feed，算法有一定扩散 |
| Article | 长篇 thought leadership，有 SEO 价值 | 主页展示，搜索可发现 |

---

## Text Post（纯文字）

**适用：** 行业观点、个人洞察、品牌故事、轻量公告

**字数：**
- 理想：150 字以内（互动率最高）
- 可接受：150–500 字（thought leadership）
- 超过 1,300 字考虑用 Article

**文案结构：**
```
[第一行 — hook，只显示 2–3 行后折叠，这行决定是否展开]

[正文 — 每 1–2 句换行，不用密集段落]
[观点 / 数据 / 故事 / 洞察]

[CTA — 「你怎么看？」「评论区聊聊」「关注获取更多」]

[Hashtag — 3–5 个，放在文末]
```

**写作要点：**
- 第一行是最关键的，不能浪费在客套话上
- **不要在正文放链接**，LinkedIn 算法会降低触达；把链接放第一条评论
- 每 1–2 句强制换行，LinkedIn 不支持密集段落
- 用「你」和读者对话，而不是「我们公司……」

---

## Image Post（图文）

**适用：** 产品推广、活动照片、数据图表、团队照

**图片规格：**
- 张数：1–9 张（多张时以相册形式展示）
- 比例：1:1（1200×1200）/ 16:9（1200×627）
- 格式：JPG / PNG，最大 5MB

**文案结构：**
```
[第一行 hook]

[正文 — 为图片提供背景/语境，不重复图片内容]

[CTA]

[Hashtag — 3–5 个]
```

**写作要点：**
- LinkedIn 图片不如 IG 那么「美观驱动」，更偏向信息价值
- 带真实人物的图片（团队照、活动照）互动率高于纯产品图
- 同样：链接放评论，不放正文

**配图 aspect_ratio：** `"1:1"` 或 `"16:9"`

---

## Document Carousel（PDF 文档轮播）

**适用：** 行业知识、品牌介绍、产品目录、教程步骤、年度总结

**规格：**
- 格式：PDF 上传（非图片 Carousel）
- 页数：最多 300 页，建议 **8–12 页**效果最好
- 每页建议只有一个核心观点

**文案结构（caption）：**
```
[第一行 hook — 说明这份文档解决什么问题]

[1–2 句正文 — 预告内容，引导点击「翻阅」]

[CTA — 「翻阅完整版 →」「保存这篇」「转发给需要的朋友」]

[Hashtag — 3–5 个]
```

**agent 需要额外生成「分页大纲」（每页标题 + 内容要点）：**
```
封面（第 1 页）：[主标题 + 副标题，决定是否继续翻]
第 2 页：[观点 / 步骤 1 — 标题 + 2–3 行说明]
第 3 页：[观点 / 步骤 2]
...
最后一页：[总结 + CTA + 品牌信息（logo、联系方式）]
```

**写作要点：**
- 封面是最关键的，要像一本书的封面一样吸引人打开
- 每页文字控制在 30 字以内，大字标题 + 简短说明
- Document Carousel 停留时长高，LinkedIn 算法偏好
- 保存率（save）是重要互动指标，在 CTA 里引导「保存」

---

## Video（视频）

**适用：** 产品演示、客户案例、幕后故事、活动回顾

**视频规格：**
- 时长：3s–10 分钟，建议 **1–3 分钟**
- 比例：1:1 / 16:9 / 9:16 均支持
- 格式：MP4，最大 5GB

**文案结构：**
```
[第一行 hook]

[正文 — 视频背景介绍，不剧透结论]

[CTA]

[Hashtag — 3–5 个]
```

**写作要点：**
- LinkedIn 视频自动静音播放，前 3 秒要有字幕或视觉 hook
- 建议加字幕（accessiblity 且更多人无声观看）
- 链接同样放评论

**配图 aspect_ratio：** `"16:9"` 或 `"1:1"`

---

## Article（长文章）

**适用：** Thought leadership、深度行业分析、SEO 内容

**特性：**
- 独立的文章编辑器（非普通 feed post）
- 有标题、封面图、正文（支持 H2/H3/图片/引用）
- 发布后在主页「文章」区展示，有 SEO 可发现性
- 字数不限，建议 800–2,000 字

**agent 生成内容（仅生成大纲 + 正文，不走普通发布流程）：**
```
标题：[吸引点击的标题]
副标题 / 摘要：[1–2 句，搜索结果显示]
封面图建议：[图片描述]
正文结构：
  H2：[章节 1 标题]
  [内容]
  H2：[章节 2 标题]
  [内容]
  ...
  结尾 CTA：[引导关注 / 评论 / 联系]
```

**注意：** Article 需要在 LinkedIn 平台手动排版发布，publishing skill 不处理 Article。

---

## 最佳发布时间

| 类型 | 推荐时间 |
|---|---|
| Text / Image / Video | 周二、三、四，早 8–10 点 |
| Document Carousel | 周二、三，早 9–10 点（工作日开始时） |
| Article | 周二、四，早 8–9 点 |
