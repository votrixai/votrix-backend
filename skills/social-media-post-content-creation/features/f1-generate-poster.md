# F1 — 图片素材生成海报（设计哲学驱动版）

**核心工具**：Python Pillow（AI 生成脚本，沙盒运行）

---

## 步骤 1：收集信息

确认以下信息，缺什么问什么，齐了再继续：

- **商家图片**：本地路径或公开 URL（JPG / PNG）
- **必须出现在海报上的文字**：标题、价格、卖点、地址、联系方式等，逐一确认
- **目标平台**：决定海报尺寸
  - Instagram Feed / 小红书：1080×1080 或 1080×1350
  - Instagram Story / 抖音 / 微信朋友圈竖版：1080×1920
  - 未指定默认：1080×1080
- **品牌色**（可选）：未提供则由 AI 根据图片主色调 + 设计哲学推导
- **风格偏好**（可选）：用户可说"高端""活泼""简约""日系"等关键词

---

## 步骤 2：读取已有海报设计哲学

检查 `/workspace/brand-style/poster-philosophy.md` 是否存在：

- **存在** → 读取其中的美学方向、色彩系统、排版气质、构图方向，作为本次设计的基础参考，保持品牌一致性。用户若有新的风格偏好可在此基础上调整。
- **不存在** → 继续步骤 3，从图片自主推导设计哲学，完成后写入该文件。

---

## 步骤 3：图片分析

**在形成设计哲学之前，先"读懂"这张图。**

观察/推断以下要素：
- **主体位置**：主体偏上 / 偏下 / 居中 / 偏左 / 偏右
- **图片明暗**：整体偏亮 / 偏暗 / 对比强烈
- **色调倾向**：暖色（橙红黄）/ 冷色（蓝绿紫）/ 中性
- **内容类型**：食物特写 / 空间全景 / 人物 / 产品静物 / 户外风景
- **留白区域**：哪些区域相对空旷可以放文字
- **图片情绪**：奢华 / 亲切 / 清新 / 力量 / 温柔 / 都市感

---

## 步骤 4：制定设计哲学

**这是最关键的一步，决定整张海报的灵魂。** 在选择任何布局或颜色之前，先为这张海报命名一个美学方向，并明确三个维度。

### 3a. 命名美学方向（1–3个词）

根据图片情绪 + 用户风格偏好 + 商品品类，为这张海报定义一个美学方向。例如：

- **"暖调工匠感"**：食物/手作类，温暖、质感、用心
- **"极简都市"**：时尚/科技类，克制、留白、高级
- **"热带丰收"**：生鲜/农产品类，饱和、活力、自然
- **"深夜静谧"**：咖啡/酒吧类，暗调、质感、诗意
- **"阳光街角"**：早餐/花店类，明亮、清新、生活感
- **"奢宴仪式"**：高端餐饮/礼品类，金色、庄重、仪式感

这个名字不展示给用户，只作为内部设计决策锚点。

### 3b. 色彩哲学

基于美学方向 + 图片主色调，推导出一套色彩系统：

- **主色调**：支配整体气氛的颜色（文字、色块、背景）
- **辅助色**：呼应主色的第二层颜色（分割线、标签底色）
- **点睛色**：少量使用的高对比强调色（价格、关键词）

**原则**：颜色不从图片里"挑"出来，而是与图片"对话"——图片已有的色彩是背景，设计的颜色是回应。色彩系统应该整体和谐，不超过3种主要颜色。

### 3c. 排版气质

根据美学方向选择字体组合和排版态度：

**气势型**（促销、价格冲击、街头感）：
- 标题字体：`BigShoulders-Bold.ttf` 或 `EricaOne-Regular.ttf` 或 `Boldonse-Regular.ttf`
- 副标题：`WorkSans-Bold.ttf` 或 `BricolageGrotesque-Bold.ttf`

**精致型**（高端、奢华、轻奢）：
- 标题字体：`Gloock-Regular.ttf` 或 `Italiana-Regular.ttf` 或 `IBMPlexSerif-Bold.ttf`
- 副标题：`InstrumentSans-Regular.ttf` 或 `Jura-Light.ttf`

**现代型**（简约、都市、科技感）：
- 标题字体：`Outfit-Bold.ttf` 或 `WorkSans-Bold.ttf` 或 `BricolageGrotesque-Bold.ttf`
- 副标题：`InstrumentSans-Regular.ttf` 或 `Outfit-Regular.ttf`

**温柔型**（生活感、日系、小清新）：
- 标题字体：`Lora-Bold.ttf` 或 `CrimsonPro-Bold.ttf` 或 `YoungSerif-Regular.ttf`
- 副标题：`Lora-Regular.ttf` 或 `InstrumentSerif-Regular.ttf`

**点缀字体**（少量用于装饰性文字、英文标语、slogan）：
- `NothingYouCouldDo-Regular.ttf`（手写感）
- `PoiretOne-Regular.ttf`（轻盈优雅）
- `ArsenalSC-Regular.ttf`（小型大写，精致感）

**中文文字必须使用**：`/usr/share/fonts/truetype/wqy/wqy-zenhei.ttc`
**英文/数字字体路径**：`../canvas-design/canvas-fonts/{字体文件名}`

### 3d. 空间构图方向

根据美学方向和图片主体位置，选择空间构图策略：

- **沉浸型**：图片铺满，文字嵌入留白或压暗区域，呼吸感强
- **分区型**：图片与文字区域清晰分隔，信息密度高
- **叠层型**：图片作底，多层半透明色块营造纵深感
- **框架型**：图片居中展示，周围留边做排版舞台

构图策略决定下一步的 Pillow 实现。

**设计哲学确定后，立即写入 `/workspace/brand-style/poster-philosophy.md`**，格式如下：

```
# 海报设计哲学

## 美学方向
[命名的美学方向]

## 色彩系统
- 主色调：[颜色 + hex]
- 辅助色：[颜色 + hex]
- 点睛色：[颜色 + hex]

## 排版气质
- 标题字体：[字体文件名]
- 副标题字体：[字体文件名]
- 点缀字体：[字体文件名（如有）]

## 空间构图
[沉浸型 / 分区型 / 叠层型 / 框架型 + 一句说明]
```

## 步骤 5：生成 Pillow 合成脚本

根据步骤 3 的设计哲学，直接生成完整 Python 脚本。构图、布局、层次均由设计哲学推导，不套用固定模板。

### 通用规范（所有布局都遵守）：

**背景层：**
- 加载商家图片，按目标尺寸 `scale + center crop` 填满，不留黑边，不拉伸
- 模糊程度根据布局需要调整

**文字规范：**

| 层级 | 内容示例 | 字号参考 |
|-----|---------|---------|
| 最大 | 价格 / 核心数字 | 80–160px |
| 大 | 主标题 / 品名 | 48–64px |
| 中 | 副标题 / 卖点 | 28–36px |
| 小标签 | 卖点词组 | 26–32px |
| 最小 | 联系 / 地址 | 22–28px |

**标签组排列**：横向流式，间距 16px，超出宽度自动换行。每个标签左右 padding 12–16px，上下 6–8px。标签样式（描边/填充/药丸）根据美学方向选择。

**安全边距**：所有文字和元素距画布边缘 ≥ 40px。

**输出**：保存为 PNG 至 `/mnt/user-data/outputs/poster_{slug}.png`

### 工匠标准

每一张海报都应当看起来像是由顶级设计师耗费数小时精心打磨的作品。具体要求：

- **间距精确**：文字间距、元素间距不靠感觉，用数值保证一致性
- **色彩克制**：严格遵守步骤3的色彩系统，不随意引入第四种颜色
- **字体混搭有节制**：标题、副标题、标签各用一种字体，最多三种，保持视觉统一
- **每个细节都有理由**：分割线的粗细、圆角的大小、遮罩的透明度，都应服务于设计哲学
- **没有元素是多余的**：如果某个元素去掉不影响信息传达和美感，就不要加
- **检查视觉重心**：整体构图是否稳定？眼睛的移动路径是否自然？

AI 被鼓励在此框架内自由发挥：分割线样式、圆角大小、遮罩形状、文字对齐、装饰细线等。但每个选择都要服务于设计哲学，而非随机变化。

---

## 步骤 6：沙盒执行

在沙盒中运行脚本。若报错，根据错误信息修正后重试，最多 3 次。

常见错误处理：
- `FileNotFoundError` → 检查图片路径；URL 先用 urllib/requests 下载到本地
- `OSError: cannot open resource` → 中文字体路径写死为 `/usr/share/fonts/truetype/wqy/wqy-zenhei.ttc`；英文字体确认 `../canvas-design/canvas-fonts/` 路径正确
- 输出文件为空 → 检查 `canvas.save()` 是否执行

---

## 步骤 7：质量复查

**分两阶段进行。有明显问题则修正脚本重新执行，最多 3 轮。**

---

### 阶段一：看图前——写下设计意图

在读取图片之前，根据步骤 3 的设计哲学，先写下以下预期：

- **照片**：应占画面多少比例？主体是否应清晰可见？
- **标题**：视觉重量最重的文字是哪一句？它应该在画面哪个位置？
- **层级**：第一眼看到什么，第二眼看到什么？
- **点睛色**：出现在哪些元素上？是否有节制地复用？
- **整体情绪**：陌生人 3 秒扫完，应该感受到什么？

---

### 阶段二：看图后——对照检查

读取输出 PNG，先与阶段一的预期逐一对照，再做逐项检查：

**与设计意图对照：**
- 照片是否如预期可见，还是被遮罩/色块掩盖了？
- 最重要的信息是否在第一眼就清晰突出？
- 点睛色是否克制地复用，而非只出现一次或到处乱用？
- 整体情绪是否与美学方向一致？

**逐项合格检查：**

| 检查项 | 合格标准 |
|-------|---------|
| 文字完整 | 步骤 1 收集的所有信息都出现在图片上 |
| 无截断 | 没有文字被裁到画布外 |
| 无乱码 | 中文字符正常显示，无 □□□ |
| 可读性 | 所有文字与背景有足够对比度，尤其是小字 |
| 信息不冗余 | 同一信息没有在两处重复出现 |
| 标签未溢出 | 标签组在画布宽度范围内 |
| 图片未变形 | 背景图比例正常，无拉伸，核心主体未被裁切 |
| 视觉统一 | 各区块（顶部/中部/底部）色调和气质一致，无割裂感 |
| 哲学一致性 | 颜色、字体、构图忠实于步骤 3 的设计哲学 |
| 工匠水准 | 间距均匀、颜色克制、没有多余元素、视觉重心稳定 |

**判断标准：** 以上任意一项有明显问题（不是「可以更好」，而是「用户看到会皱眉」），就必须修正并重做。

3 轮后仍有问题 → 向用户说明具体是哪一项问题、当前无法自动修复的原因，请求进一步指导。

---

## 步骤 8：生成文案

为海报配套 social media caption。

**原则**：
- Caption 补充海报上没有的信息（背景故事、使用体验、为什么选择）
- 不重复海报已有的文字
- 结尾引导行动：私信 / 扫码 / 电话咨询

**Hook 参考（选一种）**：
- 数字冲击：`[面积/价格等数字] + 出乎意料的卖点`
- 场景代入：`如果你也在找……`
- 直接价值：`[地点] + [核心卖点] + 现在开放`

**Hashtag**：5–12 个，覆盖大众标签 + 垂直标签 + 地理/品牌标签

---

## 输出

向用户展示：
1. 海报图片（使用 present_files 工具）
2. 所用美学方向 + 一句话说明设计决策
3. Caption + Hashtags
4. 询问：满意直接发布，还是想换一种风格试试？
