# LinkedIn Publishing API

---

## Prerequisite: Obtain Author URN

An author URN is needed before publishing. Read it from `## Connected Platforms → LinkedIn` in `/workspace/marketing-context.md`.

- Personal account: `urn:li:person:{person_id}`
- Company page: `urn:li:organization:{org_id}`

---

## Text Post (text only)

```
LINKEDIN_CREATE_LINKED_IN_POST(
  author      = "urn:li:organization:{org_id}",
  commentary  = {body text, line break every 1–2 sentences, max 3000 characters},
  visibility  = "PUBLIC",
  lifecycleState = "PUBLISHED"
)
→ data = ugcPost URN (post_id, write to post-history)
```

**Notes:**
- Do not put links in the body text; after publishing, place links in the first comment (see "Link Handling" below)

---

## Image Post (image + text)

**Must follow a two-step process** -- first use `upload_file` to upload the image to Composio's S3, then create the post:

```
Step 1 — Upload image to Composio S3:
upload_file(
  file_path    = "/mnt/session/outputs/{image filename}",  # must be in outputs/
  toolkit_slug = "linkedin",
  tool_slug    = "LINKEDIN_CREATE_LINKED_IN_POST"
)
→ Returns {s3key, name, mimetype}

Note: Images generated by image_generate are in /mnt/session/uploads/, must first copy to outputs/:
bash(command='cp /mnt/session/uploads/{generated_xxx.jpeg} /mnt/session/outputs/{filename.jpeg}')

Step 2 — Create post, using the s3key, name, and mimetype returned from Step 1 in images:
LINKEDIN_CREATE_LINKED_IN_POST(
  author         = "urn:li:organization:{org_id}",
  commentary     = {body text},
  images         = [{s3key: "{s3key}", name: "{name}", mimetype: "{mimetype}"}],
  visibility     = "PUBLIC",
  lifecycleState = "PUBLISHED"
)
→ data = ugcPost URN (post_id, write to post-history)
```

**Note:** The images parameter only accepts `{s3key, name, mimetype}`; URLs or URNs cannot be used.

## Link Handling

LinkedIn's algorithm reduces reach for posts containing external links in the body text. Handle as follows:

```
Step 1 — Publish the post body (without links)

Step 2 — Immediately after publishing, post a comment under the post, placing the link in the first comment:
LINKEDIN_CREATE_COMMENT_ON_POST(
  actor      = "urn:li:organization:{org_id}",
  object     = {ugcPost_urn},   # The just-published post URN
  target_urn = {ugcPost_urn},
  message    = {text: "Link: {url}"}
)
```

## Error Handling

| Error | How to Handle |
|---|---|
| Token expired / insufficient permissions | Inform admin they need to reconnect LinkedIn, guide them to run setup |
| 429 rate limit | LinkedIn shared OAuth app has strict rate limits; suggest retrying later, or prompt admin to use their own OAuth credentials |
| Body exceeds 3000 characters | Truncate to 3000 characters, inform admin |
| upload_file failed | Check if file_path is in /mnt/session/outputs/; images from image_generate need to be copied to outputs/ first |
| LINKEDIN_CREATE_LINKED_IN_POST image 404 | Confirm the images parameter uses the s3key returned by upload_file, not a URL or URN |
| Content violates platform policy | Return LinkedIn's original error, suggest modifying and retrying |
