# Twitter / X Publishing API

---

## Image Upload (only available method)

Twitter supports image publishing (up to 4 images per tweet, each ≤ 5 MB, supports JPEG / PNG / WEBP).

**Must strictly follow these four steps; none can be skipped:**

```
Step 0 — Confirm image is in /mnt/session/outputs/ (only needed for image_generate output):
  image_generate output is in /mnt/session/uploads/, must first copy to /mnt/session/outputs/:
  bash: cp /mnt/session/uploads/{filename} /mnt/session/outputs/{filename}
  Images generated by poster-design are already in /mnt/session/outputs/, skip this step.

Step 1 — Upload to Composio storage (must execute first to obtain s3key):
upload_file(
  file_path    = "/mnt/session/outputs/{filename}",   # cannot be a /mnt/session/uploads/ path
  toolkit_slug = "twitter",
  tool_slug    = "TWITTER_UPLOAD_MEDIA"
)
→ {s3key, name, mimetype}   ← This is a Composio internal s3key, not a Supabase URL

Step 2 — Upload to Twitter, obtain media_id:
TWITTER_UPLOAD_MEDIA(
  media          = {s3key, name, mimetype},   # The complete return value from Step 1, passed as-is
  media_category = "tweet_image"
)
→ data.media_id  (pure numeric string, e.g. "1782345678901234567")

Step 3 — Post tweet with image:
TWITTER_CREATION_OF_A_POST(
  text            = {tweet body, ≤280 characters},
  media_media_ids = ["{media_id}"]
)
→ data.id = tweet_id
```

**Key restrictions:**
- Step 1 cannot be skipped: `TWITTER_UPLOAD_MEDIA` only accepts Composio s3key, not Supabase URLs
- `media_id` expires very quickly; execute Step 3 immediately after Step 2
- Maximum 4 images: execute Steps 0–2 for each image separately, collect all `media_id` values, then pass them all at once in Step 3

---

## Video Content Limitation

Not supported. When the draft type is "Twitter video", handle with fallback: if there is an image, publish with the image attached; if no image, generate one first then publish with it attached.

## Single Tweet

```
TWITTER_CREATION_OF_A_POST(
  text            = {tweet body, ≤280 characters, links count as 23 characters},
  media_media_ids = ["{media_id}"]   # Optional, media_id must be obtained through the four-step process above
)
→ data.id = tweet_id (write to post-history)
```

**Notes:**
- Hashtags go directly in `text` (at most 1–2)
- Links go at the end of `text`; regardless of actual URL length, they always count as 23 characters

## Thread

Threads are chained via `reply_in_reply_to_tweet_id`, published sequentially:

```
Step 1 — Post the 1st tweet (hook):
TWITTER_CREATION_OF_A_POST(
  text = {1st tweet copy, ≤280 characters}
)
→ tweet_id_1

Step 2 — Post the 2nd tweet (reply to the 1st):
TWITTER_CREATION_OF_A_POST(
  text                       = {2nd tweet copy, ≤280 characters},
  reply_in_reply_to_tweet_id = {tweet_id_1}
)
→ tweet_id_2

... and so on, each tweet replies to the previous one

Last tweet: summary + CTA
```

**Notes:**
- Each tweet must be ≤280 characters (links count as 23 characters)
- Must be posted in order; you need the previous tweet_id before posting the next one
- Only record the 1st tweet's tweet_id in post-history

---

## Link Handling

Twitter links go directly at the end of the body text; regardless of the original URL length, they always count as 23 characters. No special handling needed.

---

## Error Handling

| Error | How to Handle |
|---|---|
| Token expired | Inform admin they need to reconnect Twitter, guide them to run setup |
| Tweet exceeds 280 characters | Check `text` character count, count links as 23 characters, truncate and retry |
| `TWITTER_UPLOAD_MEDIA` 404 / s3key invalid | Confirm `upload_file` was called first, use the s3key it returned, do not use Supabase URL |
| Non-numeric string in `media_media_ids` | Check that media_id is a pure number (not a placeholder), re-obtain and retry |
| `media_media_ids` invalid | media_id may not have gone through TWITTER_UPLOAD_MEDIA, or may have expired; re-execute Steps 1–2 |
| Content violates platform policy | Return Twitter's original error message, suggest modifying and retrying |
| Publishing failed (network/rate limit) | Keep draft, suggest retrying later |
